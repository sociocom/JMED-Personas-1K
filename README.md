# データセット概要 (Dataset Overview)

NAIST-Patient-Personas(NPP1000) は、日本語の合成患者ペルソナJSONLデータセットです。公開統計・補助リソースから構成した確率分布と制約条件をもとに、合成患者の基本属性、疾患情報、検査・治療情報、生活背景、生活習慣、性格特性、ペルソナ記述を生成しています。

本データセットは、医療AI/ヘルスケアLLMの研究・評価、プロンプト検証、デモ用途での利用を想定しています。実在患者の診療録、問診票、検査結果、個人情報を収集または変換したものではありません。

各JSONLは1行が1人分の合成患者レコードです。本READMEでは、主に `data/persona_demo_1000_offline_vllm_no_catalog_clean_v3.jsonl` を説明対象とします。このファイルは，傷病分類までは固定条件として与えていますが、主病名、ICDコード、診療科、症状、検査、処方、治療方針などはLLMが生成しています。

## 使用モデル

患者ペルソナ本文と、固定条件以外の構造化項目の生成には、[GPT-OSS-120B / openai/gpt-oss-120b](https://huggingface.co/openai/gpt-oss-120b) を使用しています。固定条件、JSON Schema、プロンプトを与え、vLLM の offline batched inference で1患者1JSONオブジェクトとして生成しています。

## 生成方法と項目の生成元

`data/persona_demo_1000_offline_vllm_no_catalog_clean_v3.jsonl` では、すべての項目を同じ方法で生成しているわけではありません。機械的に決めた値を固定条件としてLLMへ渡し、LLMが残りの構造化項目とペルソナ文章を補完する構成です。

| 生成元 | 対象項目 | 説明 |
| --- | --- | --- |
| プログラム付与 | `患者ID` | UUID形式の合成レコード識別子として後処理で付与します。LLMには生成させません。 |
| 機械的に選択・固定 | `都道府県`, `郵便番号`, `年齢`, `性別`, `傷病分類`, `入院／外来`, `同居／独居`, `生活支援・介護資源`, `喫煙歴`, `飲酒歴`, `運動習慣`, `睡眠`, `BIG FIVE性格特性` | 統計・補助リソースとseed付きサンプリングで事前に決めます。LLM出力後も固定値に補正されます。 |
| LLM生成の構造化項目 | `患者ID`、上記の固定項目、`ペルソナ_*` を除く項目 | 固定条件、JSON Schema、プロンプトを入力として、医学情報、検査・治療情報、生活背景、安全評価、発話などを生成します。 |
| LLM生成の文章項目 | `ペルソナ_*` 10項目 | 構造化項目の内容を根拠として、患者像を文章化します。患者名は独立項目ではなく、ペルソナ文章の主語としてのみ含まれます。 |

## 生成に用いた資料

NPP1000 は、以下の資料・リソースから作成した確率分布と制約条件をもとに生成しています。

- [厚生労働省 令和5年患者調査](https://www.mhlw.go.jp/toukei/saikin/hw/kanja/23/index.html): 年齢層、性別、傷病分類の分布制約に使用。
- [e-Stat 令和5年患者調査 全国編 Z16-1](https://www.e-stat.go.jp/stat-search/files?cycle=7&layout=datalist&page=1&stat_infid=000040234267&tclass1=000001224322&tclass2val=0&toukei=00450022&tstat=000001224321): 患者調査由来の補助資料として、傷病分類や診療設定の検討に使用。
- [e-Stat 令和5年国民健康・栄養調査](https://www.e-stat.go.jp/stat-search/files?cycle=7&layout=datalist&tclass1=000001228532&tclass2val=0&toukei=00450171&tstat=000001041744): 喫煙歴、飲酒歴、運動習慣、睡眠、身体計測の制約に使用。
- [日本郵便 UTF-8郵便番号データ](https://www.post.japanpost.jp/service/search/zipcode/download/utf/zip/utf_ken_all.zip): `resources/japan_postal_codes_by_prefecture.json` の都道府県別郵便番号候補に使用。
- [nvidia/Nemotron-Personas-Japan](https://huggingface.co/datasets/nvidia/Nemotron-Personas-Japan): 日本語姓名候補としてペルソナ文章の主語に使用。


## データセット詳細 (Dataset Details)

JSONLの各行には71項目が含まれます。そのうち非ペルソナ項目は61項目、ペルソナ文章項目は10項目です。

主な項目カテゴリは次のとおりです。

- 基本属性: `患者ID`, `都道府県`, `郵便番号`, `年齢`, `性別`
- 社会・リテラシー: `教育歴`, `医療・健康リテラシー`, `職業`, `デジタルリテラシー・端末利用`, `経済的制約`
- 医学情報: `傷病分類`, `主病名`, `ICDコード`, `診療科`, `入院／外来`, `症状`, `既往歴`, `家族歴`, `感染症歴`
- 検査・治療: `身長`, `体重`, `BMI`, `バイタルサイン`, `診察所見`, `血液検査`, `画像検査`, `生理検査`, `処方薬`, `治療方針・治療目標`
- 生活背景: `同居／独居`, `生活支援・介護資源`, `居住環境`, `医療アクセス・通院手段`, `ADL/IADL`
- 生活習慣: `喫煙歴`, `飲酒歴`, `運動習慣`, `普段の食生活`, `睡眠`
- 心理・性格: `BIG FIVE性格特性`, `趣味・大切にしている活動`, `患者の語り/代表発話`
- ペルソナ文章: `ペルソナ_基本属性`, `ペルソナ_医学的背景`, `ペルソナ_受診に至った経過`, `ペルソナ_診察・検査所見`, `ペルソナ_治療・服薬上の課題`, `ペルソナ_生活背景・支援状況`, `ペルソナ_価値観・心理面`, `ペルソナ_生活習慣`, `ペルソナ_支援ポイント`, `ペルソナ_統合記述`

### 非ペルソナ項目の説明

#### 基本属性

| 項目 | 解説 |
| --- | --- |
| `患者ID` | 合成レコードを識別するUUID形式のIDです。実在患者や外部IDとは対応しません。 |
| `都道府県` | 合成患者の居住都道府県です。市区町村以下の住所は含めません。 |
| `郵便番号` | 都道府県に対応する郵便番号候補から選ばれた値です。個人住所を示すものではありません。 |
| `年齢` | 合成患者の年齢です。患者調査由来の年齢層分布をもとに機械的に選ばれます。 |
| `性別` | 合成患者の性別カテゴリです。現在のスキーマでは `男性` または `女性` のいずれかです。 |

#### 社会・リテラシー

| 項目 | 解説 |
| --- | --- |
| `教育歴` | 最終学歴または年齢に応じた教育段階を示します。小児や学生では年齢と矛盾しない表現を想定します。 |
| `医療・健康リテラシー` | 健康情報の理解、評価、意思決定への利用しやすさを示します。本人だけでなく家族や支援者の関与も含めた設定です。 |
| `職業` | 現在の就労状況や退職後の前職などを示します。個人特定につながる勤務先名は含めません。 |
| `デジタルリテラシー・端末利用` | スマートフォンやPCなどの利用状況と、医療情報検索への慣れを示します。年齢だけで一律に決めず、個人差を持たせています。 |
| `経済的制約` | 医療費、交通費、生活費などに関する制約の有無や程度を示します。制度利用や受診継続の支援ニーズを表す背景情報です。 |
| `妊娠・授乳/生殖関連` | 妊娠、授乳、閉経、妊娠可能性など、診療上関連し得る情報を示します。該当しない場合も明示的な値を入れます。 |

#### 医学情報

| 項目 | 解説 |
| --- | --- |
| `傷病分類` | 患者調査由来の大分類で、機械的に選択される固定項目です。`no_catalog` 生成では、この分類に合う具体疾患をLLMが生成します。 |
| `主病名` | レコードの中心となる具体的な疾患名または受診目的です。固定された傷病分類とICD章に矛盾しないようLLMが生成します。 |
| `ICDコード` | 主病名に対応するICD-10コードです。`no_catalog` 生成では疾患カタログ固定ではなく、主病名と傷病分類に合わせてLLMが生成します。 |
| `診療科` | 主病名や診療設定に対応する診療科です。疾患、検査、治療方針と整合するように生成されます。 |
| `入院／外来` | 外来、入院、救急外来からの入院、訪問診療、施設内診療などの診療設定です。分布設定に基づき機械的に固定されます。 |
| `受診経路` | 自己受診、紹介、救急搬送、施設内紹介など、受診に至った経路を示します。固定された診療設定に合わせてLLMが記述します。 |
| `かかりつけ・医療利用状況` | かかりつけ医の有無、健診、定期受診、救急利用などの医療利用背景を示します。継続受診や情報連携の文脈に使います。 |
| `疾患ステージ・重症度` | 疾患の進行度、重症度、リスク段階などを示します。検査所見、治療方針、診療設定と整合することを意図しています。 |
| `受診理由・主訴` | 患者または家族が受診を決めた主要な訴えです。症状だけでなく健診異常や生活上の困りごとも含みます。 |
| `現病歴` | 症状や異常の出現から受診までの経過を示します。時系列、悪化要因、受診判断の背景を含めます。 |
| `症状` | 主病名に関連する主要症状を短く列挙します。診察所見や検査内容と対応することを想定しています。 |
| `既往歴` | 併存疾患や過去に診断された重要な疾患を示します。処方、禁忌、治療目標に影響する背景情報です。 |
| `家族歴` | 血縁者の医学的リスクや関連疾患の有無を示します。同居状況や支援者情報とは区別します。 |
| `感染症歴` | 結核、肝炎、HIVなど、診療上重要な感染症歴を示します。特記すべき事項がない場合も明示します。 |

#### 検査・治療

| 項目 | 解説 |
| --- | --- |
| `身長` | 合成患者の身長です。年齢、性別、体重、BMIと大きく矛盾しない値を想定しています。 |
| `体重` | 合成患者の体重です。身長、BMI、疾患背景と整合するように生成されます。 |
| `BMI` | 身長と体重に対応する体格指数です。`kg/m2` 表記で記載されます。 |
| `バイタルサイン` | 体温、血圧、脈拍、呼吸数、SpO2などの基本的な生体情報です。疾患や重症度を反映する臨床情報として生成されます。 |
| `診察所見` | 身体診察や専門診察で得られる所見を示します。主病名、症状、検査と整合することを意図しています。 |
| `血液検査` | 血算、生化学、炎症反応、血糖、腎機能などの検査情報です。疾患や処方判断に関わる値を含みます。 |
| `画像検査` | X線、CT、MRI、超音波、内視鏡、眼科画像などの画像所見を示します。疾患に応じて不要な場合は `なし` などで示します。 |
| `生理検査` | 心電図、肺機能、視力検査、神経生理検査などの機能検査を示します。現行データでは一部に培養検査などの欄違い候補があります。 |
| `健診歴` | 過去の健康診断や定期検査の受診状況を示します。疾患発見の契機や医療利用状況の背景になります。 |
| `処方薬` | 主病名や併存疾患に対する薬剤、用量、頻度などを示します。現行データには薬剤用量や表記の確認候補があります。 |
| `服薬管理・アドヒアランス` | 服薬を本人、家族、施設職員などがどの程度管理できているかを示します。服薬忘れや自己中断などの課題も含みます。 |
| `サプリメント` | 使用しているサプリメントの有無や種類を示します。処方薬との併用や生活背景の補助情報です。 |
| `アレルギー` | 薬剤、食品、環境などのアレルギー情報を示します。禁忌事項や処方判断と関連します。 |
| `禁忌事項` | 明確な禁忌や強い注意が必要な条件を示します。該当がない場合は明示的に記載します。 |
| `手術歴` | 過去の手術経験を示します。主病名や治療方針に関連する場合があります。 |
| `使用している医療機器` | 杖、車椅子、酸素機器、補聴器などの使用状況を示します。移動、安全、生活支援の文脈に関わります。 |
| `治療方針・治療目標` | 薬物療法、手術、リハビリ、生活指導などの方針と目標を示します。主病名、重症度、生活背景に合わせてLLMが生成します。 |

## 確認済みissue / 今後修正すべき点

以下は `data/persona_demo_1000_offline_vllm_no_catalog_clean_v3.jsonl` と既存の監査結果を読んで確認した修正候補です。これらは機械監査と目視確認に基づく候補であり、医学的な確定判定ではありません。

- `受診経路` の表記ゆれが多く、86種類の値が残っています。`自己受診`、`外来予約`、`紹介外来`、`救急搬送`、`施設内診療` などの標準カテゴリへ正規化する必要があります。
- `嚥下機能` と `妊娠・授乳/生殖関連` に自由記述、句点つき表現、同義表現が混在しています。前者は35種類、後者は34種類の値があり、カテゴリ値と説明文の分離が必要です。
- 65歳未満、`ADL/IADL` が `自立`、`認知・意思決定能力` が `問題なし` にもかかわらず、`訪問介護` や `デイサービス` が付く候補が55件あります。支援理由を本文に明記するか、支援区分を `家族支援中心` や `なし` などへ修正する必要があります。
- 65歳以上で `親と同居` となる例が24件あります。特に75歳以上の例は年齢上かなり不自然になりやすいため、`子と同居`、`配偶者と同居`、`その他` などへの修正候補として優先確認が必要です。
- 軽症・慢性・スクリーニング系疾患に対して、入院や救急受診が設定されている確認候補が残っています。受診経路監査では96件の確認候補があり、急性増悪や合併症で説明できるかを個別に確認する必要があります。
- 肺結核治療でピラジナミド欠落が疑われる例、痰培養や抗酸菌検査などが `生理検査` 欄に入っている例、H. pylori除菌レジメンの用量・期間が不自然な可能性のある例があります。疾患別テンプレートまたは薬剤用量表に基づく修正が必要です。
- `シタグリプチン10mg`、`シロスタゾール10mg`、`エナラプリル50mg`、`インスリン0.5単位` などの薬剤用量疑義があります。腎機能調整や診療文脈を確認したうえで、標準用量に照らした修正が必要です。
- `アトorvastatin` など、日本語薬剤名に英字が混入した表記候補があります。薬剤名の正規表記表を用意し、`アトルバスタチン` などへ一括正規化することが望まれます。
